Published Wednesday, May 8, 1996, in the Miami Herald.

Reader straightened out `host' question

Last week, I answered a question about dialing from one home computer to another by using the simple Windows Terminal communications program on each machine. I said it couldn't be done because Terminal doesn't have the ``host'' mode necessary for handling incoming calls, that you had to have more sophisticated communications software that includes a ``host'' mode.

Well, I was wrong.

Eighteen-year-old Alan Goldstein of Tamarac gently pointed out to me that it can be done.

Here's how: Have Terminal open on both computers. On Computer No. 1 (the one receiving the call), type ``ATA'' on the screen. Next, have Computer No. 2 dial Computer No. 1. Once the phone rings at Computer No. 1 (you have to have a telephone on the same line as the computer, which isn't unusual), hit ``enter'' or ``return.''

The ``ATA'' command tells Computer No. 1 to answer the call. It should pick up the phone. Then, the two computers are connected. People at the two keyboards can communicate by typing on the screen. Characters typed will show up on the other computer, as well.

To transfer a file between computers, start on the computer that will send the file. Choose ``Transfer/Send,'' and then give the file name to send. On the receiving end, choose ``Transfer/Receive,'' and give a name that the file will have when it arrives.

This isn't as smooth as using a ``host'' mode on fancier software, but it definitely will do the trick.

Goldstein says he isn't an electronic guru.

``I have friends who are computer wizards,'' he says. ``I'm just a computer operator. I know my limits.''

So does CyberGuide.


Q.
I'm running Windows95 and showing only 504K of conventional memory. I need more. I could run memmaker on my old Windows 3.1, but not here. Any ideas on how to increase my conventional memory?

-- J.P., South Florida


A.
Memmaker is still your friend. Windows95 doesn't need it for running Windows programs, but it is still handy for DOS programs, especially games. You can keep your memmaker command in your autoexec.bat file. Windows95 will ignore it during bootup, but will start it when it loads a virtual DOS machine to run a DOS program.

Also, you can also load system programs in high memory to free up the 640K conventional memory. On Windows95, you need to edit the ``msdos.sys'' file in the ``c:\'' directory. Before you can edit it, you must remove the read-only protection.

At the MSDOS prompt, type ``attrib -r-s-h msdos.sys'' and hit enter.

Then, edit the file using Notepad. Add the line: ``LoadTop=1'' and save.

Put the read-only protection back on by typing ``attrib +r+s+h msdos.sys'' and hit enter.

Restart the computer. You should have more conventional memory for your DOS program.
